# Specs Guide

- Keep `specs/mission.md`, `specs/tech-stack.md`, and `specs/roadmap.md` as
  the planning source documents.
- Keep later roadmap concerns out of the current phase unless specs are revised
  and approved.
- For phase-specific work, inspect and edit only the active or affected phase
  folder by default. Other phase folders are out of scope unless the user asks
  for a cross-phase change, the roadmap/stable docs require it, or an approved
  amendment targets that phase.
- Leave blocked, partial, or unvalidated work unchecked.
